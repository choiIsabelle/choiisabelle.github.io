About my Work Process...

Determine Needs üîé

Before determining the implementation and design of a website, I consider the specific use cases of a website, its target audience, and key stakeholders. I look for relevant existing products to determine how I can map existing design patterns to my current project. I prioritize key features and begin considering an initial design.

Conceptualize Solution üìù

Using the initial idea I have constructed based on the needs of the user, I begin to collect existing design ideas and draft multiple potential website designs using Figma. I consider the user experience and user interface of the website, and how I can best implement the design in a way that is modular and reusable. I also consider how I can best implement the website using modern technologies.

Implementing A Solution üíª

Now that I have chosen a specific design, I determine the technical solutions (programming language, framework, libraries) that allow me to best produce my solution whilst accelerating my development workflow. I begin with a top-down approach and establish different architectural layers before writing any code. I regularly test the functionality of my website as I work and constantly verify I am meeting the needs of the project.
